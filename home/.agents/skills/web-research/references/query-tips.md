# Query Formulation Tips

- Be specific -- include library names, version numbers, and language
- Use domain-specific terms, not generic descriptions
- Include the year for evolving topics: `"Next.js 15 app router 2025"`
- Separate queries per comparison option when evaluating alternatives
- Quote exact error messages when researching errors
- Add `"official docs"` or `"documentation"` to target authoritative sources

## Examples

Good: `"zod v3 discriminated union validation TypeScript"`
Bad: `"how to validate data in typescript"`

Good: `"RuntimeError: dictionary changed size during iteration python 3.12"`
Bad: `"python dictionary error"`
